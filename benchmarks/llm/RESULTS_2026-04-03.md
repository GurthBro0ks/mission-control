# Local LLM Benchmark Results (CPU-only)

Date: 2026-04-03 UTC  
Host: slimy-nuc1  
Runtime: Ollama 0.12.11  
CPU: Intel i5-6260U (4 threads)  
RAM: 15 GiB total (~10 GiB available during test)  
GPU: none

## Test design

- API: `POST /api/generate` (Ollama)
- Prompt: `Write exactly 120 words explaining why unit tests matter in production software.`
- Output target: `num_predict=120`
- Threads: `num_thread=4`
- Repetitions: 3 runs/model
- Metric used for sizing decisions: generation throughput (`eval_count / eval_duration`)

Notes:
- Prompt eval throughput varied heavily because runs after the first benefitted from warm cache/load effects.
- Generation throughput is the most stable and relevant metric for interactive chat speed.

## What this server can run well

- Best interactive quality/speed zone: **0.5B to 1B** instruct models.
- Usable but slower: **3B** models (good for background tasks, slower for chat).
- Works but noticeably sluggish: **7B** models (expect ~30-45s for ~120-token answers).

## Practical recommendation

For this exact machine (CPU-only, 4 threads), use:
- Primary chat model: `llama3.2:1b` or `qwen2.5:0.5b`
- Higher-quality offline jobs: `qwen2.5:3b`
- Avoid for interactive UX: `7B+` models unless latency is acceptable
